■苦労した点

・ナビメッシュの構築に苦労しました。初めは、ポリゴンの頂点にWayPointを配置していたのですが、ポリゴンの形に依存したギザギザな経路になってしまったので、頂点と頂点の中間地点にWayPointを配置することで、滑らかな経路にすることができました。ただ、この場合ですと、WayPoint同士を接続する際に、モデルを貫通して接続してしまう問題がありましたので、ポリゴンの法線情報から、適切な接続相手を判断することで、モデルとの当たり判定をして貫通するかの判定を省き、処理速度が遅くならないように、適切なナビメッシュ構築を実現しました。

■アピールポイント

・開発の中で、WayPointを座標を指定して、配置するのがとても大変だったので、ナビメッシュを導入して、どのようなステージにも自動でWayPointを配置できるようにすることで、作業時間の短縮とナビゲーションが導入しやすいように改善したした。

・敵の経路の複数化とユーザー体験に驚きをもたせるために、壁を登る敵を導入し、ナビメッシュを使って、壁にもWayPointを配置し、経路探索を導入しました。

・敵が最適な経路を選択するように、敵の攻撃力と遮断物の体力を見て、遮断物を破壊して進むコストと迂回して進むコストを比較して、最短経路を導くシステムを導入することで、ステージ状況に対応した、賢い敵を追加しました。

・処理負荷軽減のために、ステージ状況が変化した時のみ、動的に敵が進む経路を更新し変化した範囲のみのナビメッシュの再計算で、動的な経路探索を高速に処理するよう工夫しました。

・ゲームのデータ調整をゲーム内で完結させるために、JSONを使った書き出しと読み込みをImGuiで実行して、ゲーム内でデータを変更できるように実装しました。

■タイトル

 「  DEFENCE CORE  」

■ゲーム内容

「罠を設置して敵からコアを守るゲーム」です。

プレイヤーは俯瞰視点で罠の設置、一人称視点では、敵の行動を遅くするスライムを発射できます。

- 罠の種類 -
防御壁  	：敵の移動ルート上に配置することで、敵の経路を遮断できます。
スパイク	：敵の移動ルート上に配置することで、通った敵にダメージを与えることができます。
タレット	：タレットの視界に入った敵を攻撃します。弾がなくなると警告パップアップが出てきますので、弾の補充が必要となります。
ジャンプパッド	：プレイヤーが乗ることで、遠くへジャンプ移動することができます。

敵からコアを守ることができれば勝利、守れなければ敗北となります。

■実装内容

- システム -
・A*アルゴリズムで経路探索を導入し、敵が目的地に進む実装をしました。
・ナビメッシュによる、モデル情報からWayPointを自動配置するシステムを導入し、ステージが変わってもすぐに対応できるようにしました。
・様々な敵の進行するルートを導入するために、壁を登る敵用のWayPoint配置をナビメッシュを使って実装しました。
・動的ナビゲーションによる、敵がステージの状況変化（罠の配置などによる変化）に対応し、経路を変更できるようにしました。
・経路が遮断された際に、迂回するコストと遮断物の破壊コストを比較し、最適な経路を選択する敵AIを実装しました。
・JSONを活用した外部化をしました。

- ゲーム -
・複数の罠の実装しました。
・一人称視点と俯瞰視点を実装しました。
・俯瞰視点での罠の設置を実装しました。
・タレット罠の強化、リロードを実装しました。
・敵の行動を遅くするスライムの発射を実装しました。
・敵を攻撃する弾の発射を実装しました。
